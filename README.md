# E-Commerce Sales Data Analysis Using SQL

## Project Overview
This project analyzes an e-commerce sales dataset using SQL to extract valuable business insights. 
The analysis focuses on revenue generation, customer behavior, product performance, and order trends.

The project demonstrates the use of SQL concepts such as:
- Joins
- Aggregation Functions
- Window Functions
- Ranking Functions
- Subqueries

---

## Dataset Description

The project uses four tables:

### Customers
customer_id  
customer_name  
city  

### Products
product_id  
product_name  
category  
price  

### Orders
order_id  
customer_id  
order_date  

### Order Items
order_id  
product_id  
quantity  

---

## Database Relationships

customers.customer_id = orders.customer_id  

orders.order_id = order_items.order_id  

products.product_id = order_items.product_id  

---

## SQL Problems Solved

1. Find the total revenue generated by each product category.  
2. Identify the customer who generated the highest revenue.  
3. Retrieve the top 3 most sold products based on quantity.  
4. Find the total number of orders placed by each customer.  
5. Identify customers who have never placed an order.  
6. Find the order that generated the highest revenue.  
7. Calculate cumulative revenue by order date.  
8. Identify the most expensive product in each category.  
9. Find the latest order placed by each customer.  

---

## Example SQL Query

```sql
SELECT p.category,
SUM(oi.quantity * p.price) AS revenue
FROM order_items oi
JOIN products p
ON oi.product_id = p.product_id
GROUP BY p.category;
